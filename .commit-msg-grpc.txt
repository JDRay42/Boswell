Phase 2 Progress: gRPC service layer implementation

Implements Stream A of Phase 2 - gRPC API surface for Boswell.

Protocol Definitions (proto/boswell.proto):
- AssertRequest/Response with tier targeting and provenance support
- QueryRequest/Response with QueryMode (fast/deliberate) and filters
- LearnRequest/Response for bulk claim insertion (ADR-012)
- ForgetRequest/Response for eviction marking
- HealthCheckRequest/Response with status, version, uptime
- Common types: Tier enum, ConfidenceInterval, Claim, ProvenanceEntry

Service Implementation:
- BosWellServiceImpl with Arc<Mutex<S: ClaimStore>>
- assert() - validates auth, converts proto types, stores claims
- query() - builds ClaimQuery, applies filters (namespace, tier, confidence, subject/predicate/object)
- learn() - bulk insertion with error collection
- forget() - claim lookup and eviction marking (stub for Phase 3)
- health_check() - returns service status and metrics
- Error mapping with Debug trait bounds for ClaimStore::Error
